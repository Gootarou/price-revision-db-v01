# 手動テストチェックリスト

Phase 7 時点では、v0.1を実運用テストに入れる前の手動確認項目を整理する。Googleスプレッドシート上でメニュー操作とシート内容を確認する。

## 初期セットアップ

- [ ] 料金改定DB メニューが表示される
- [ ] 初期セットアップを実行できる
- [ ] 必要シートが作成される
- [ ] ヘッダーが作成される
- [ ] シート順が想定通りになる
- [ ] 管理_処理ログ にログが残る
- [ ] データ_料金改定案件 の条件付き書式が設定される
- [ ] 確認_文書転記前 の条件付き書式が設定される

## 入力画面初期化

- [ ] 入力_案件編集 を初期化できる
- [ ] 固定列エラーが出ない
- [ ] A1:J1 のタイトル結合が正常に表示される
- [ ] 人間入力セルが薄黄色になる
- [ ] 自動採番・自動計算・自動判定予定セルが薄グレーになる
- [ ] セル保護が実装されていない
- [ ] 現行単位がプルダウンになり、月額が初期値になる
- [ ] 業務種別がプルダウンになり、日常清掃が初期値になる
- [ ] 端数処理方式がプルダウンになり、なしが初期値になる
- [ ] 使用有無がプルダウンになり、使用が初期値になる
- [ ] 年間休日を入れると、勤務日数/年 が 365 - 年間休日 になる
- [ ] 勤務日数/年、労働時間/日、人員/日 がそろうと、年間総労働時間 が計算される
- [ ] 必要な値がない場合、勤務日数/年 や 年間総労働時間 は空欄になる
- [ ] 入力_案件編集 の計算結果表示エリアに条件付き書式が設定される
- [ ] 使用有無 = 不使用 の勤務時間根拠行が不使用色になる

## 保存

- [ ] 必須項目を入力して保存できる
- [ ] サービスID が採番される
- [ ] 案件ID が採番される
- [ ] マスタ_契約サービス に保存される
- [ ] データ_料金改定案件 に保存される
- [ ] 明細_勤務時間根拠 に保存される
- [ ] 入力画面の 年間総労働時間 が 明細_勤務時間根拠 に値として保存される
- [ ] 空の勤務時間根拠行は保存されない
- [ ] 既存案件IDで保存した場合、重複せず更新される
- [ ] 保存時に確認状態が未確認に戻る

## 読込

- [ ] 案件IDを指定して読込できる
- [ ] 基本情報が入力画面に反映される
- [ ] 日付・料金条件が入力画面に反映される
- [ ] 勤務時間根拠が入力画面に反映される
- [ ] 既存の勤務時間根拠明細値が壊れない
- [ ] 既存明細がない空行には補助数式と使用有無初期値が設定される
- [ ] 計算結果表示エリアが反映または空欄化される
- [ ] 勤務時間根拠が表示可能行数を超える場合、警告またはエラーになる

## 保存して計算

- [ ] 月額案件を保存して計算できる
- [ ] 計算結果が データ_料金改定案件 に保存される
- [ ] 計算結果が 入力_案件編集 に表示される
- [ ] 最低賃金が 前回改定開始日・今回改定開始日 から参照される
- [ ] 計算係数が 今回改定開始日 から参照される
- [ ] 使用有無 = 使用 の勤務時間根拠だけ集計される
- [ ] 端数処理が 暫定月額増加額 に対して行われる
- [ ] 手動調整額が反映される
- [ ] 文書転記OK が判定される
- [ ] 正常計算結果が条件付き書式追加前後で変わらない

## 対象外・エラー

- [ ] 現行単位 = 年額 の場合、対象外になる
- [ ] 現行単位 = 日額 の場合、対象外になる
- [ ] 現行単位 = 回 の場合、対象外になる
- [ ] 最低賃金未登録時にエラーになる
- [ ] 計算係数未登録時にエラーになる
- [ ] 勤務時間根拠未登録時にエラーになる
- [ ] 年間総労働時間が空欄の場合にエラーになる
- [ ] 年間総労働時間が空白文字だけの場合に0扱いされずエラーになる
- [ ] 前回改定開始日と今回改定開始日の逆転ケースを確認する
- [ ] 計算結果状態 = エラー のセルがエラー色になる
- [ ] 確認状態 = 未確認 のセルが未確認色になる
- [ ] 文書転記OK = NG のセルがNG色になる
- [ ] エラー内容が空欄でない場合に警告色になる

- [ ] 案件情報が表示される

- [ ] 不一致の場合、警告が表示される
- [ ] 別シートで実行するとエラーになる
