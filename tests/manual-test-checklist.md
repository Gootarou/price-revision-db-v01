- [ ] キャンセルを選ぶと入力画面が変更されない
- [ ] 入力_案件編集 を初期化できる
- [ ] 固定列エラーが出ない
- [ ] A1:J1 のタイトル結合が正常に表示される
- [ ] 人間入力セルが薄黄色になる
- [ ] 自動採番・自動計算・自動判定予定セルが薄グレーになる
- [ ] セル保護が実装されていない
- [ ] 現行単位がプルダウンになり、月額が初期値になる
- [ ] 業務種別がプルダウンになり、日常清掃が初期値になる
- [ ] 端数処理方式がプルダウンになり、なしが初期値になる
- [ ] 使用有無がプルダウンになり、使用が初期値になる
- [ ] 年間休日を入れると、勤務日数/年 が 365 - 年間休日 になる
- [ ] 勤務日数/年、労働時間/日、人員/日 がそろうと、年間総労働時間 が 勤務日数/年 × 労働時間/日 × 人員/日 で計算される

- [ ] 入力_案件編集 の計算結果表示エリアに条件付き書式が設定される
- [ ] 使用有無 = 不使用 の勤務時間根拠行が不使用色になる

- [ ] 既存案件IDで保存した場合、重複せず更新される
- [ ] 保存時に確認状態が未確認に戻る

## 読込

- [ ] 案件読込を実行すると、読込前に確認ダイアログが表示される
- [ ] 文言が「現在の入力内容を上書きして、案件を読み込みます。実行しますか？」になっている
- [ ] OKを選ぶと案件読込が実行される
- [ ] キャンセルを選ぶと案件読込が実行されない
- [ ] 案件IDを指定して読込できる
- [ ] 基本情報が入力画面に反映される
- [ ] 日付・料金条件が入力画面に反映される
- [ ] 勤務時間根拠が入力画面に反映される
- [ ] 案件読込後も 勤務日数/年 と 年間総労働時間 の補助数式が残る
- [ ] 案件読込後に 年間休日、労働時間/日、人員/日を変更すると 勤務日数/年 と 年間総労働時間 が再計算される
- [ ] 勤務日数/年 は 年間休日 から算出され、年間休日を105日にすると260日になる
- [ ] 既存の勤務時間根拠明細値が壊れない
- [ ] 保存後、明細_勤務時間根拠 には 勤務日数/年 と 年間総労働時間 が値として保存される
- [ ] 既存明細がない空行には補助数式と使用有無初期値が設定される
- [ ] 計算結果表示エリアが反映または空欄化される
- [ ] 勤務時間根拠が表示可能行数を超える場合、警告またはエラーになる

## 保存・計算実行

- [ ] メニュー名が「保存・計算実行」になっている
- [ ] 保存・計算実行を実行すると、入力内容が保存され、計算結果が更新される
- [ ] 成功時に「保存し、計算結果を更新しました。」と表示される
- [ ] 月額案件を保存・計算実行できる
- [ ] 計算結果が データ_料金改定案件 に保存される
- [ ] 計算結果が 入力_案件編集 に表示される
- [ ] 最低賃金が 前回改定開始日・今回改定開始日 から参照される
- [ ] 計算係数が 今回改定開始日 から参照される
- [ ] 使用有無 = 使用 の勤務時間根拠だけ集計される
- [ ] 端数処理が 暫定月額増加額 に対して行われる
- [ ] 端数処理対象が 暫定月額増加額 のままであり、最終提案料金そのものには適用されない
- [ ] 端数処理の結果が従来と変わらない
- [ ] 手動調整額が端数処理後月額増加額に加算される
- [ ] 月額増加額 × 12 で年間増加額が計算される
- [ ] 月額案件で現行時間単価が 現行年額換算 ÷ 年間総労働時間_明細合計 で計算される
- [ ] 月額案件で最終提案時間単価が 提案年額換算 ÷ 年間総労働時間_明細合計 で計算される
- [ ] 月額案件で時間単価差額が 最終提案時間単価 - 現行時間単価 で計算される
- [ ] 文書転記OK が判定される
- [ ] 時間単価系が文書転記OK判定に影響しない
- [ ] 文書転記OK判定が時間単価系追加前後で変わらない
- [ ] 正常計算結果が条件付き書式追加前後で変わらない

## 対象外・エラー

- [ ] 現行単位 = 年額 の場合、対象外になる
- [ ] 現行単位 = 日額 の場合、対象外になる
- [ ] 現行単位 = 回 の場合、対象外になる
- [ ] 最低賃金未登録時にエラーになる
- [ ] 計算係数未登録時にエラーになる
- [ ] 勤務時間根拠未登録時にエラーになる
- [ ] 計算エラー時に時間単価系（現行時間単価、最終提案時間単価、時間単価差額）が空欄化される
- [ ] 計算対象外時に時間単価系（現行時間単価、最終提案時間単価、時間単価差額）が空欄化される
- [ ] 年間総労働時間が空欄の場合にエラーになる
- [ ] 年間総労働時間が空白文字だけの場合に0扱いされずエラーになる
- [ ] 前回改定開始日と今回改定開始日の逆転ケースを確認する
- [ ] 計算結果状態 = エラー のセルがエラー色になる
- [ ] 確認状態 = 未確認 のセルが未確認色になる
- [ ] 文書転記OK = NG のセルがNG色になる
- [ ] エラー内容が空欄でない場合に警告色になる

## 勤務時間根拠の運用ルール

- [ ] 曜日違い・複数パターンを勤務時間根拠の行分けで入力できる
- [ ] 平日、土曜、月1回など、勤務日数・労働時間・人員が異なる条件を別行で登録できる
- [ ] 特殊要件や算出理由を 根拠メモ に残せる
- [ ] 特殊な勤務日数は直接上書き列ではなく 年間休日 の調整で表現できる
- [ ] 保存後、明細_勤務時間根拠 には補助数式ではなく値として保存される
- [ ] 勤務時間根拠入力エリアの行追加が未実装であることを前提に、表示可能行数の範囲で確認する
