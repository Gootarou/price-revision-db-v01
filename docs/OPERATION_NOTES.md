# 運用メモ

## 基本運用フロー

1. マスタ類を整備する
2. 入力_案件編集 を初期化する
3. 案件情報を入力する
4. 勤務時間根拠を入力する
5. 一時保存する
6. 保存・計算実行する
7. 必要に応じて勤務時間根拠サイドバーで確認する
8. 文書転記前確認を更新する
9. 確認_文書転記前 を見ながらWord等へ手動転記する


## メニュー操作と確認ダイアログ（v0.1.1相当）

* 一時保存は、入力内容を正本シートへ保存する操作です。料金改定計算は実行しません。
* 保存・計算実行は、入力内容を正本シートへ保存したうえで、料金改定計算を実行し、計算結果を更新します。
* 一時保存、保存・計算実行は、実行前確認ではなく、成功後に完了通知を表示します。
  * 一時保存の成功時は「一時保存しました。計算は実行していません。」と表示します。
  * 保存・計算実行の成功時は「保存し、計算結果を更新しました。」と表示します。
* 案件読込は、現在の入力画面を正本シートの内容で上書きするため、実行前に確認ダイアログを表示します。
* 入力画面初期化は、現在の入力内容を消去するため、実行前に確認ダイアログを表示します。
* 未保存変更の自動検知、入力内容の差分比較、確認済操作は未実装です。

## 注意事項

* 入力_案件編集 は正本ではない。
* 確認_文書転記前 は正本ではない。
* サイドバーは確認専用であり、編集・保存・計算実行には使わない。
* 計算対象は月額のみ。
* 年額、日額、回、㎡、その他など月額以外は現時点では自動計算対象外。
* 最低賃金参照年月は人間が入力しない。
* 最低賃金は日付から自動参照する。
* 勤務時間根拠の詳細は 明細_勤務時間根拠 に保存する。
* データ_料金改定案件 に勤務時間根拠の詳細列を増やさない。
* 時間単価系（現行時間単価、最終提案時間単価、時間単価差額）は 年間総労働時間_明細合計 を分母にする。
* 年間総労働時間_明細合計 が正しくない場合、時間単価系の計算結果も正しくならない。
* 時間単価系は内部確認用・計算結果確認用であり、外部向け交渉資料には使わない。
* 時間単価系は現時点では 文書転記OK 判定の必須条件ではない。

## 勤務時間根拠の入力運用

* 勤務時間根拠は、1行で無理に表現せず、勤務日数、労働時間、人員が異なる単位で行分けする。
* 曜日ごと、作業頻度ごと、繁忙期・閑散期など条件が異なる場合は、勤務時間根拠を行で分けて年間総労働時間を積み上げる。
  * 例：平日、土曜、日曜、週1回、月1回、繁忙期、閑散期。
* 勤務日数/年 は、基本的に 年間休日 を調整して算出する。
  * 例：実際の勤務日数を260日にしたい場合は、年間休日を105日にして 365 - 105 = 260 とする。
* 特殊な勤務日数や算出理由は 根拠メモ に残す。
  * 例：工場カレンダーにより年間勤務日数260日で算出。
  * 例：週1回作業のため年間52日で算出。
  * 例：実稼働日数に合わせるため年間休日を調整。
  * 例：月1回作業のため年間12日で算出。
* 現時点では、勤務日数/年 や 年間総労働時間 の直接上書き用列は追加しない。
* 補助数式は 入力_案件編集 上で維持し、正本である 明細_勤務時間根拠 には保存時点の値として保存する。

## 入力補助（v0.1.1相当）

* 入力_案件編集 の以下項目は、入力画面初期化時にプルダウンが設定されます。
  * 現行単位：月額 / 年額 / 日額 / 回 / ㎡ / その他（初期値：月額）
  * 業務種別：日常清掃 / 定期清掃 / その他（初期値：日常清掃）
  * 端数処理方式：なし / 10円未満切り下げ / 100円未満切り下げ / 1000円未満切り下げ（初期値：なし）
  * 使用有無：使用 / 不使用（初期値：使用）
* 勤務時間根拠入力エリアの 勤務日数/年 と 年間総労働時間 は、入力画面上の補助数式です。
  * 年間休日が入力されている場合、勤務日数/年 は 365 - 年間休日 になります。
  * 勤務日数/年、労働時間/日、人員/日 がそろっている場合、年間総労働時間 が計算されます。
  * 必要な値がない場合は空欄になります。
  * 既存案件を案件読込した後も、入力_案件編集 上では 勤務日数/年 と 年間総労働時間 を補助数式として維持します。
* 明細_勤務時間根拠 には、保存時点の入力画面の値として保存されます。明細_勤務時間根拠 へ補助数式は保存しません。
* 例外的な勤務日数や年間総労働時間を使いたい場合は、現時点では年間休日、労働時間/日、人員/日側で調整します。
* 使用有無の意味は以下の通りです。
  * 使用：計算対象に含めます。
  * 不使用：明細として残しますが、計算対象外にします。
  * 削除：勤務時間根拠入力エリアから行内容を空にして保存すると、データとして残しません。

## 条件付き書式（v0.1.1相当）

* 初期セットアップ、入力画面初期化、文書転記前確認ビュー更新で、状態確認用の条件付き書式が再適用されます。
* 色の意味は以下の通りです。
  * 薄赤：エラー、文書転記NG、エラー内容あり
  * 薄黄：未確認、保留
  * 薄緑：OK、確認済、承認
  * 薄グレー：対象外、不使用、見送り
  * 薄青：交渉中
* 条件付き書式は確認補助です。保存・読込・料金計算・文書転記OK判定のロジックは変更しません。

## 計算エラー時に確認する代表シート

* 現場ID・都道府県エラーの場合は、入力_案件編集、データ_料金改定案件、マスタ_現場 を確認する。
  * 現場IDが未入力の場合は、入力_案件編集 または データ_料金改定案件 の現場IDを入力し、マスタ_現場に都道府県を登録する。
  * マスタ_現場に現場IDが存在しない場合は、マスタ_現場に現場ID・現場名・都道府県を登録する。
  * マスタ_現場の都道府県が未入力の場合は、都道府県を入力し、マスタ_最低賃金の都道府県表記と一致させる。
* 最低賃金エラーの場合は、マスタ_最低賃金 を確認し、同じ都道府県で基準日以前の適用開始日と最低賃金を登録する。
* 計算係数エラーの場合は、マスタ_計算係数 を確認し、基準日以前の適用開始日と法定福利費計算倍率を登録する。
* 勤務時間根拠エラーの場合は、入力_案件編集、明細_勤務時間根拠 を確認し、使用有無が「使用」の行と年間総労働時間を登録する。

## 日付・タイムゾーン運用

* GASプロジェクトのタイムゾーンは Asia/Tokyo を推奨する。
* スプレッドシートのタイムゾーンも日本時間を推奨する。
* v0.1の料金改定計算では、今回改定開始日・前回改定開始日・マスタの適用開始日は日付のみとして扱う。時刻差による誤判定を避けるため、コード上のエラーメッセージ表示とマスタ適用日の比較は日本時間の日付キーで判定する。
