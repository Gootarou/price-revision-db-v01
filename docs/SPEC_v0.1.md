# 料金改定計算DB 改良版 v0.1 詳細仕様書

## 1. システム概要

本システムは、料金改定に必要な根拠データを入力・編集し、最低賃金上昇額ベースの提案料金を計算し、人間が確認したうえで、Word等へ手動転記できる状態まで管理するためのGoogleスプレッドシート + Google Apps Scriptアプリである。

v1では文書自動生成を行わない。
v1は料金改定計算・確認DBである。

## 2. 基本方針

* 旧システムの番号付きシート名は使わない。
* 通常操作の入口は 入力_案件編集 に集約する。
* 入力_案件編集 は正本ではない。
* 正本となる主データは データ_料金改定案件 と 明細_勤務時間根拠 である。
* 契約先、現場、契約サービス、最低賃金、計算係数はマスタとして管理する。
* 確認_文書転記前 は正本ではなく参照ビューである。
* AppSheet化はv1では実装しない。
* 文書生成はv1では実装しない。
* 状態別の色変更はGASで実装しない。

### 6.3 採番ルール

* 管理_設定 に年度別連番を保持する。
* 新規保存時に採番する。

## 9. 日付仕様

日付はスプレッドシートの日付型で保持する。
表示形式は yyyy/mm/dd を推奨する。

| 日付項目    | 役割          | 最低賃金参照      |
| ------- | ----------- | ----------- |
| 前回改定開始日 | 前回料金が始まった日  | 前回参照最低賃金に使用 |
| 今回改定依頼日 | 顧客へ依頼・提出した日 | 原則使用しない     |
| 今回改定開始日 | 今回新料金を始めたい日 | 今回参照最低賃金に使用 |

## 10. 最低賃金参照仕様

1. 対象案件の現場IDを取得する。
2. マスタ_現場 から現場IDに一致する都道府県を取得する。
3. マスタ_最低賃金 から都道府県が一致する行を抽出する。
4. 基準日以前の適用開始日を持つ行を抽出する。
5. その中で最も新しい適用開始日の最低賃金を採用する。

基準日：

| 参照対象     | 基準日     |
| -------- | ------- |
| 前回参照最低賃金 | 前回改定開始日 |
| 今回参照最低賃金 | 今回改定開始日 |

## 11. 計算係数参照仕様

1. 今回改定開始日を基準日とする。
2. マスタ_計算係数 から、基準日以前の適用開始日を持つ行を抽出する。
3. その中で最も新しい適用開始日の法定福利費計算倍率を採用する。
4. 該当がない場合はエラーとする。

## 12. 計算仕様

### 12.1 自動計算対象

v1の自動計算対象は 現行単位 = 月額 の案件のみとする。

年額、日額、回、㎡、その他など、月額以外は現時点では自動計算対象外とする。月額以外の場合は、計算結果状態を 対象外 とし、エラー内容に v1自動計算対象外 と記録する。月額以外の自動計算対応は、今回追加しない。

### 12.2 月額契約の計算順

1. 前回参照最低賃金を取得する。
2. 今回参照最低賃金を取得する。
3. 最低賃金上昇額 = 今回参照最低賃金 - 前回参照最低賃金
4. 法定福利費計算倍率を取得する。
5. 労務単価上昇額 = 最低賃金上昇額 × 法定福利費計算倍率
6. 明細_勤務時間根拠 の 使用有無 = 使用 の年間総労働時間を合計する。
7. 暫定年間増加額 = 年間総労働時間_明細合計 × 労務単価上昇額
8. 暫定月額増加額 = 暫定年間増加額 ÷ 12
9. 端数処理後月額増加額 = 暫定月額増加額に端数処理方式を適用
10. 月額増加額 = 端数処理後月額増加額 + 手動調整額
11. 年間増加額 = 月額増加額 × 12
12. 現行月額換算 = 現行料金
13. 現行年額換算 = 現行月額換算 × 12
14. 最終提案料金 = 現行月額換算 + 月額増加額
15. 提案月額換算 = 最終提案料金
16. 提案年額換算 = 提案月額換算 × 12
17. 現行時間単価 = 現行年額換算 ÷ 年間総労働時間_明細合計
18. 最終提案時間単価 = 提案年額換算 ÷ 年間総労働時間_明細合計
19. 時間単価差額 = 最終提案時間単価 - 現行時間単価

時間単価系（現行時間単価、最終提案時間単価、時間単価差額）は、年間総労働時間_明細合計を分母として計算する。内部値は表示用に丸めず、表示丸めはスプレッドシートの表示形式に任せる。

時間単価系は データ_料金改定案件 に保存し、入力_案件編集 に表示する。確認_文書転記前 には内部確認用として表示してよい。ただし、内部確認用・計算結果確認用に限定し、外部向け交渉資料には使用しない。また、文書転記OK判定条件には含めない。

### 12.3 端数処理

| 端数処理方式      | 処理             |
| ----------- | -------------- |
| なし          | 暫定月額増加額をそのまま使用 |
| 10円未満切り下げ   | 10円単位に切り下げ     |
| 100円未満切り下げ  | 100円単位に切り下げ    |
| 1000円未満切り下げ | 1000円単位に切り下げ   |

端数処理は、提案総額ではなく 暫定月額増加額 に対して行う。手動調整額は端数処理後月額増加額に加算し、月額増加額を算出する。

端数処理方式は現状維持とし、切り上げ、四捨五入、最終提案料金そのものへの端数処理、端数処理方式の追加は行わない。

### 13.3 交渉状況

| 値   |
| --- |

## 14. 文書転記OK判定

以下をすべて満たす場合に OK とする。

* 計算結果状態 = OK
* 確認状態 = 確認済
* エラー内容が空欄
* 最終提案料金が空欄でない
* 提案月額換算が空欄でない
* 提案年額換算が空欄でない
* 今回改定開始日が空欄でない
* 年間総労働時間_明細合計 が0より大きい

それ以外は NG とする。

現行時間単価、最終提案時間単価、時間単価差額は確認用の計算結果として保存・表示するが、現時点では 文書転記OK 判定条件には含めない。

## 15. 勤務時間根拠仕様

### 15.1 正本

勤務時間根拠の正本は 明細_勤務時間根拠 とする。

### 15.2 編集方針

通常編集は 入力_案件編集 から行う。
明細_勤務時間根拠 を直接編集する運用は保守用とする。

### 15.3 計算対象

使用有無 = 使用 の明細のみ計算対象とする。

### 15.4 明細計算

勤務日数/年 = 365 - 年間休日
年間総労働時間 = 勤務日数/年 × 労働時間/日 × 人員/日

勤務日数/年 は、基本的に 年間休日 を調整して算出する。特殊な勤務日数を使う場合も、現時点では直接上書き用の列を追加しない。年間休日を調整して算出し、算出理由や特殊要件は 根拠メモ に残す。

入力_案件編集 上では 勤務日数/年 と 年間総労働時間 を補助数式として維持する。明細_勤務時間根拠 には保存時点の値を保存し、数式は保存しない。

### 15.5 行分け運用

勤務時間根拠は、1行で無理に表現しない。勤務日数、労働時間、人員が異なる場合は、行を分けて登録する。曜日ごと、作業頻度ごと、繁忙期・閑散期など複数パターンがある場合も、行ごとに年間総労働時間を積み上げる。

行分けの例：

| 区分 | 年間休日 | 勤務日数/年 | 労働時間/日 | 人員/日 | 根拠メモ例 |
| --- | ---: | ---: | ---: | ---: | --- |
| 平日 | 105 | 260 | 2.0 | 1 | 工場カレンダーにより年間勤務日数260日で算出 |
| 土曜 | 313 | 52 | 3.0 | 1 | 週1回作業のため年間52日で算出 |
| 月1回 | 353 | 12 | 1.5 | 1 | 月1回作業のため年間12日で算出 |

## 16. 勤務時間根拠サイドバー仕様

### 16.1 起動方法

データ_料金改定案件 で対象行を選択し、メニューから 勤務時間根拠を表示 を実行する。

### 16.2 処理

1. アクティブシートが データ_料金改定案件 か確認する。
2. 選択行の案件IDを取得する。
3. 明細_勤務時間根拠 から同じ案件IDの明細を抽出する。
4. 使用有無 = 使用 の年間総労働時間を合計する。
5. データ_料金改定案件 の 年間総労働時間_明細合計 と比較する。
6. サイドバーに一覧表示する。

### 16.3 表示内容

* 案件ID
* 契約先名
* 現場名
* 業務名
* 明細行番号
* 区分
* 年間休日
* 勤務日数/年
* 労働時間/日
* 人員/日
* 年間総労働時間
* 使用有無
* 根拠メモ
* 備考

### 16.4 差異警告

明細側合計とデータ側合計が一致しない場合、警告を表示する。

警告文：

明細_勤務時間根拠 の使用明細合計と データ_料金改定案件 の年間総労働時間_明細合計が一致しません。保存・計算実行を実行してください。

### 16.5 編集不可

v1ではサイドバーから編集しない。
確認専用とする。

確認_文書転記前 は、データ_料金改定案件 から文書転記に必要な列のみ抽出して再生成する。

このシートは正本ではない。
