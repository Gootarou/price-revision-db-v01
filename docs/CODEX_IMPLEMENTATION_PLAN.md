# Codex実装手順書

# 料金改定計算DB 改良版 v0.1

## 1. 実装目的

Googleスプレッドシート + Google Apps Script で、料金改定計算DB 改良版 v0.1 を新規アプリとして実装する。

本アプリは、料金改定に必要な根拠データを入力・編集し、最低賃金上昇額ベースの提案料金を計算し、人間が確認したうえで、Word等へ手動転記できる状態まで管理するためのシステムである。

v1では文書自動生成は実装しない。

## 2. 実装方針

### 2.1 新規アプリとして実装する

現行リポジトリを直接拡張しない。
旧システムの番号付きシート名も引き継がない。

旧名称例：

* 00_案件入力・編集
* 05_勤務時間根拠
* 10_年度契約・改定一覧
* 11_文書転記前確認

これらは使用しない。

### 2.2 v1は計算・確認DBに絞る

v1の目的は、文書生成ではなく、根拠入力・計算・確認・転記前チェックである。

### 2.3 実装は段階分割する

Codexには一括実装させない。
Phaseごとに小さく実装し、差分確認・動作確認を行う。

## 5. GAS実装上の基本ルール

### 5.1 シート名は定数化する

コード内にシート名文字列を直書きしない。
必ず Config.gs に定義した定数を使う。

想定定数：

```javascript
const SHEETS = {
  INPUT_CASE_EDIT: '入力_案件編集',

  DATA_PRICE_REVISION_CASE: 'データ_料金改定案件',
  DETAIL_WORK_TIME_BASIS: '明細_勤務時間根拠',
  VIEW_DOCUMENT_TRANSFER_CHECK: '確認_文書転記前',

  MASTER_SERVICE: 'マスタ_契約サービス',
  MASTER_CLIENT: 'マスタ_契約先',
  MASTER_SITE: 'マスタ_現場',
  MASTER_CODE_MAP: 'マスタ_コード対応表',
  MASTER_MIN_WAGE: 'マスタ_最低賃金',
  MASTER_COEFFICIENT: 'マスタ_計算係数',

  SYSTEM_SETTINGS: '管理_設定',
  SYSTEM_LOG: '管理_処理ログ',
};
```

### 5.2 シート順に依存しない

シート順は人間の操作性のためだけに使う。
GAS処理はシート順に依存しない。

### 5.3 見た目変更をGASで実装しない

値による文字色・セル色変更は、v0.1.1相当で追加した条件付き書式を除きGASでは実装しない。

実装禁止：

* setBackground
* setFontColor
* setFontWeight
* 状態別の色変更処理
* 条件付き書式の自動生成

視覚的な強調は、人間がスプレッドシートの条件付き書式で手動設定する。

### 5.4 onEditを多用しない

v1では onEdit による自動処理を多用しない。
基本はメニュー実行とする。

### 5.5 データ処理はまとめて行う

* getValues / setValues を使う
* セル単位の読み書きを繰り返さない
* ヘッダー名から列番号を取得する
* 列番号のベタ書きを避ける

## 10. 実装時の注意点

* コード変更はPhaseごとに小さく行う。
* 実装後は変更ファイルと確認手順を報告する。
* シート名変更に強いように定数を使う。
* ヘッダー名で列を特定する。
* 既存シートを破壊しない。
* エラー時はユーザーに分かるメッセージを出す。
* ログを残す。
* 便利そうでもv1対象外の機能は追加しない。
