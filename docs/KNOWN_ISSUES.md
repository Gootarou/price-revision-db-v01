# 既知の注意点・未解決点

この文書は、料金改定計算DB 改良版 v0.1 の実運用テスト前に共有しておく注意点と未解決点を整理するものです。

## 計算・データ面

* 現行単位 = 月額 のみ自動計算対象です。
* 年額、日額、回、㎡などは v0.1 では対象外です。
* 前回改定開始日 > 今回改定開始日 の逆転チェックはまだ実装していません。
* 勤務日数/年 と 年間総労働時間 は、入力_案件編集 上の補助数式としてのみ計算します。正本シートには保存時点の値として保存します。
* 勤務時間根拠入力画面は表示可能行数に制限があり、行追加はまだ実装していません。
* 最低賃金マスタ、計算係数マスタが未整備だと計算エラーになります。最低賃金エラーは マスタ_最低賃金、計算係数エラーは マスタ_計算係数 を確認してください。
* 現場ID・都道府県の不足による計算エラーは、入力_案件編集、データ_料金改定案件、マスタ_現場 を確認してください。
* 勤務時間根拠の不足や年間総労働時間の入力不備による計算エラーは、入力_案件編集、明細_勤務時間根拠 を確認してください。
* v0.1凍結前に、GASプロジェクトとスプレッドシートのタイムゾーンが日本時間で運用されていることを確認してください。コード上は日付表示とマスタ適用日の比較を日本時間の日付キーで扱うため、2026/04/01 がエラーメッセージ上 2026/03/31 になる疑いは表示・比較ロジック側で最小対策済みです。

## 操作面

* 案件ID検索機能はまだありません。
* 契約先名・現場名からのID逆引き/検索はまだありません。
* 入力_案件編集 の現在入力クリア機能はまだありません。
* 保存前確認ダイアログはまだありません。
* 案件読込前確認ダイアログはまだありません。
* 入力画面初期化前確認ダイアログはまだありません。
* 未保存変更の警告はまだありません。
* セル保護はまだありません。
* 条件付き書式は人間が手動で設定します。

## 運用面

* 既存データ移行は自動化していません。
* Word等への文書転記は手動です。
* Google Docs自動生成やPDF作成は未実装です。
* 確認_文書転記前 は正本ではなく、再生成される確認ビューです。
